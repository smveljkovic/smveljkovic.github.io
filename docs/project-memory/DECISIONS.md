# Decisions

## Confirmed decisions

- **Use Astro as the site framework, not Eleventy or a SPA.**  
  **Rationale:** Static-first, works well with GitHub Pages, supports content collections, Markdown/MDX, clean URLs, build-time SEO/JSON-LD, and possible future islands.  
  **Implications:** Development should continue in `/Users/stevan/Projects/smvsite-astro`; Eleventy work is historical only.  
  **Affected:** `astro.config.mjs`, `src/pages/`, `src/content.config.ts`, build/deploy workflow.  
  **Status:** Final.

- **Preserve the existing site identity and visual design during migration.**  
  **Rationale:** The site is a durable academic/research hub with a distinctive personal/scholarly identity, not a generic portfolio.  
  **Implications:** Do not adopt a full Astro theme or redesign wholesale; refactor gradually.  
  **Affected:** `src/styles/global.css`, layouts, homepage, publications, review pages.  
  **Status:** Final.

- **Canonical production domain is the apex domain.**  
  **Decision:** Use `https://stevanveljkovic.com/` as the canonical site URL.  
  **Rationale:** Stable public identity and structured-data base URL.  
  **Implications:** Astro config should use `site: "https://stevanveljkovic.com"`; canonical links/schema should use apex URLs.  
  **Affected:** `astro.config.mjs`, SEO components, JSON-LD, sitemap.  
  **Status:** Final.

- **Use clean trailing-slash URLs.**  
  **Decision:** Preserve routes such as `/publications/`, `/cv/`, `/publications/reviews/cosmic-connections/`.  
  **Rationale:** Existing URL behaviour should remain stable.  
  **Implications:** Astro should use `trailingSlash: "always"`; route files should be directory-style `index.astro` where needed.  
  **Affected:** `astro.config.mjs`, `src/pages/**/index.astro`, sitemap, internal links.  
  **Status:** Final.

- **Current preserved public route model.**  
  **Decision:** Maintain these live routes: `/`, `/publications/`, `/cv/`, `/publications/reviews/cosmic-connections/`, `/publications/reviews/christian-right-europe/`.  
  **Rationale:** They are current/generated public pages.  
  **Implications:** Do not break or rename them; only actual generated pages should be in sitemap.  
  **Affected:** `src/pages/index.astro`, `src/pages/publications/index.astro`, `src/pages/cv/index.astro`, dynamic review route.  
  **Status:** Final for current routes.

- **Generate review pages from content collection Markdown.**  
  **Decision:** Reviews should be generated by `src/pages/publications/reviews/[slug]/index.astro` from `src/content/reviews/*.md`.  
  **Rationale:** Avoid one-off `.astro` pages and one-off schema files; metadata can drive pages, publications index, sitemap, and JSON-LD.  
  **Implications:** Adding a review should require one Markdown file plus any public assets, not a new route file.  
  **Affected:** `src/content/reviews/`, dynamic review route, `ReviewLayout`, `ReviewIntro`, `ReviewByLine`, `createReviewSchema.ts`.  
  **Status:** Final architecture; implementation ongoing.

- **Do not keep static per-review pages under `src/pages`.**  
  **Rationale:** Astro static routes mask dynamic routes.  
  **Implications:** Old files such as `src/pages/publications/reviews/cosmic-connections/index.astro` must be removed/moved.  
  **Affected:** `src/pages/publications/reviews/`.  
  **Status:** Final.

- **Use `reviews` only for full generated review pages.**  
  **Decision:** Do not force bibliography-only items into the `reviews` collection.  
  **Rationale:** Some publications do not yet need local full pages; forcing them into `reviews` would create unwanted routes or awkward filtering.  
  **Implications:** Full reviews live in `src/content/reviews/`; list-only entries should live separately.  
  **Affected:** `src/content/reviews/`, future `src/content/publication-items/`, `/publications/`.  
  **Status:** Final as content model; not fully implemented.

- **Create/use a lightweight `publicationItems` collection for list-only bibliography records.**  
  **Rationale:** Allows `/publications/` to become fully data-driven without generating local pages for every publication.  
  **Implications:** `/publications/` should combine full reviews and list-only publication records.  
  **Affected:** `src/content.config.ts`, `src/content/publication-items/`, `src/pages/publications/index.astro`.  
  **Status:** Confirmed direction; implementation still needed.

- **Use Astro 6 loader-style content collections.**  
  **Decision:** Use `defineCollection`, `glob` from `astro/loaders`, and `z` from `astro/zod`.  
  **Rationale:** Current Astro-compatible pattern; avoids deprecated/incorrect imports.  
  **Implications:** Avoid duplicate `schema` keys and unnecessary `type: "content"` in loader-style collections.  
  **Affected:** `src/content.config.ts`.  
  **Status:** Final.

- **Preserve legacy/global CSS for now.**  
  **Decision:** Keep legacy/generated CSS globally in `src/styles/global.css`, imported once in `BaseLayout.astro`.  
  **Rationale:** Visual fidelity matters more than immediate CSS cleanup.  
  **Implications:** Do not casually rename old classes such as `Name`, `Headings`, `BibEntry`, `review_text`, `CVEntry`, etc.  
  **Affected:** `src/styles/global.css`, all layouts/pages.  
  **Status:** Final for migration phase; later refactor possible.

- **Use root-relative asset and internal URLs.**  
  **Rationale:** Avoid fragile deep relative paths.  
  **Implications:** Links should look like `/images/headshot-1200x630.png`, `/cv/veljkovic-cv.pdf`, `/publications/reviews/...pdf`.  
  **Affected:** Markdown frontmatter, layouts, components, publications notes, static assets in `public/`.  
  **Status:** Final.

- **Use ORCID as the canonical `Person.@id`.**  
  **Decision:** `https://orcid.org/0000-0002-2599-3227` identifies Stevan Veljkovic in JSON-LD.  
  **Rationale:** Stable scholarly identifier.  
  **Implications:** Person nodes should reuse this `@id`; include ORCID, Google Scholar, GitHub in `sameAs`.  
  **Affected:** `src/data/schema/home.ts`, `src/data/schema/cv.ts`, review schema factory.  
  **Status:** Final.

- **Reject OpenAlex profile as `sameAs`.**  
  **Rationale:** The profile was considered conflated/messy.  
  **Implications:** Do not include `https://openalex.org/A5056034517` in structured data.  
  **Affected:** Person schema.  
  **Status:** Final.

- **Use DOI URLs as primary scholarly `@id`s where available.**  
  **Rationale:** DOI URLs are stable, globally recognized identifiers.  
  **Implications:** Publisher pages can be `url` or `sameAs`, but DOI URLs should identify books/articles when present.  
  **Affected:** `createReviewSchema.ts`, future publications schema, review frontmatter.  
  **Status:** Final.

- **Treat local manuscripts/pages and DOI-published articles as distinct objects.**  
  **Rationale:** Local author manuscripts may differ materially from final published versions, especially Cosmic Connections.  
  **Implications:** Do not use false `sameAs`; use `isBasedOn` and/or `citation` to the DOI version.  
  **Affected:** Review schema factory, Cosmic Connections metadata, Christian Right metadata where relevant.  
  **Status:** Final.

- **Cosmic Connections local page is an unabridged Original Submission / Author Manuscript.**  
  **Rationale:** It is materially longer/different from the final published article.  
  **Implications:** No `sameAs` to the DOI article; include version/status; cite/link the DOI-published review.  
  **Affected:** `src/content/reviews/cosmic-connections.md`, `createReviewSchema.ts`, local PDF/page.  
  **Status:** Final.

- **Use reusable review schema factory.**  
  **Decision:** Dynamic review pages should call `createReviewSchema(review)`.  
  **Rationale:** Avoid per-review schema files and keep structured data synchronized with frontmatter.  
  **Implications:** Do not use `buildReviewSchema()` in active routes; old rich Cosmic schema remains reference only.  
  **Affected:** `src/data/schema/reviews/createReviewSchema.ts`, dynamic review route.  
  **Status:** Final; details still being refined.

- **Homepage and CV schemas remain manual one-off schemas.**  
  **Rationale:** They have distinct graph structures and should not be forced through review schema factories.  
  **Implications:** Homepage uses `homeSchema`; CV uses `cvSchema`; do not import `createReviewSchema()` there.  
  **Affected:** `src/data/schema/home.ts`, `src/data/schema/cv.ts`, `src/pages/index.astro`, `src/pages/cv/index.astro`.  
  **Status:** Final.

- **Homepage should be modeled as `AboutPage`, not generic `WebPage`.**  
  **Rationale:** The homepage functions as a personal/about hub.  
  **Implications:** Homepage JSON-LD should include `ImageObject`, `Person`, `WebSite`, and `AboutPage`.  
  **Affected:** `src/data/schema/home.ts`.  
  **Status:** Final.

- **CV page should be modeled as `ProfilePage`; CV PDF as `MediaObject`.**  
  **Rationale:** This best fits the page/PDF relationship and validator behaviour.  
  **Implications:** CV schema should avoid invalid properties previously rejected.  
  **Affected:** `src/data/schema/cv.ts`, `/cv/`, `/cv/veljkovic-cv.pdf`.  
  **Status:** Final.

- **Use dated `Role` nodes for education/affiliation periods, not dates on credentials.**  
  **Rationale:** Schema validators rejected dates on `EducationalOccupationalCredential`; roles better express periods.  
  **Implications:** Credentials represent earned degrees/diplomas; dates belong on roles under `Person.affiliation`.  
  **Affected:** CV JSON-LD.  
  **Status:** Final.

- **Do not use invalid schema properties previously rejected.**  
  **Decision:** Avoid `Person.dissertation`, `OrganizationRole.worksFor`, and credential `startDate`/`endDate`.  
  **Rationale:** Validator rejected them.  
  **Implications:** Model thesis/roles with supported alternatives.  
  **Affected:** `src/data/schema/cv.ts`, future thesis schema.  
  **Status:** Final.

- **Canonical thesis slug is `/thesis/religious-atavism-climate-crisis/`.**  
  **Rationale:** This supersedes the shorter earlier slug and better reflects the title.  
  **Implications:** Future thesis page/PDF/schema should use this slug; do not revert to `/thesis/atavism-climate-crisis/`.  
  **Affected:** future thesis page, CV schema, publications list, sitemap.  
  **Status:** Final, though route may not exist yet.

- **Christian Right reviewed work should use `editor`, not `author`, for Gionathan Lo Mascolo.**  
  **Rationale:** The book is an edited volume.  
  **Implications:** Structured data should not use `"Gionathan Lo Mascolo (ed.)"` as author; display citation may still say “ed.”.  
  **Affected:** `src/content/reviews/christian-right-europe.md`, `createReviewSchema.ts`.  
  **Status:** Final.

- **Use trusted HTML bridge fields temporarily.**  
  **Decision:** Fields such as `citationHtml`, `bylineHtml`, `originalSubmissionNote`, and `publicationList.noteHtml` may be rendered with `set:html`.  
  **Rationale:** Preserves legacy citation formatting during migration.  
  **Implications:** Only use for local/trusted content; keep semantic fields separate.  
  **Affected:** Review Markdown, publications page, `ReviewIntro`, `ReviewByLine`.  
  **Status:** Final for migration phase; may be refactored later.

- **Render JSON-LD with unescaped inline script HTML.**  
  **Decision:** Use `set:html={JSON.stringify(data, null, 2)}` inside `<script is:inline type="application/ld+json">`.  
  **Rationale:** Avoid HTML-escaped JSON-LD like `&quot;@context&quot;`.  
  **Implications:** Validate rendered page source, not TypeScript object literals.  
  **Affected:** `src/components/JsonLd.astro`, schema files.  
  **Status:** Final.

- **Only actual/generated pages should appear in the sitemap.**  
  **Rationale:** Avoid advertising planned or draft URLs.  
  **Implications:** Planned review routes should be excluded until generated and intended live.  
  **Affected:** Astro sitemap integration, `src/content/reviews/*.md`, draft filtering.  
  **Status:** Final.

- **Use GitHub Pages-compatible redirect stubs for old HTML URLs if needed.**  
  **Rationale:** GitHub Pages cannot emit arbitrary server-side 301/302 redirects.  
  **Implications:** Old HTML URLs should use static stubs with canonical link, meta refresh, `location.replace`, and fallback link; no full JSON-LD.  
  **Affected:** old `/writing.html`, `/writing/ReviewCosmicConnectionsV2.html`; possibly `public/` or Astro static pages.  
  **Status:** Final strategy; implementation unconfirmed.

- **Do not replace old PDF URLs with HTML redirect pages.**  
  **Rationale:** PDF URLs should remain PDFs unless true HTTP redirects are available.  
  **Implications:** Preserve old PDFs or later use Cloudflare/real redirects.  
  **Affected:** `/writing/ReviewCosmicConnectionsV2.pdf`, `/itinerary.pdf`.  
  **Status:** Final.

- **Cloudflare is not currently required.**  
  **Rationale:** GitHub Pages is sufficient for the present site.  
  **Implications:** Do not migrate DNS/hosting solely for current needs; Cloudflare remains an option for true redirects, headers, and `www` canonicalization.  
  **Affected:** DNS, redirects, PDF indexing headers, security headers.  
  **Status:** Final for now; may change later.

- **Homepage navigation labels and targets are fixed.**  
  **Decision:** Use Contact, Résumé, Writing, Seminars.  
  **Rationale:** Preserves existing public UX.  
  **Implications:** Résumé links to `/cv/`; Writing to `/publications/`; Seminars to `https://seminars.stevanveljkovic.com/`.  
  **Affected:** `src/pages/index.astro`, homepage layout/components.  
  **Status:** Final.

- **Accessibility fixes should be preserved.**  
  **Decision:** Internal CV link should not use `_blank`; decorative icons should have `alt="" aria-hidden="true"`; `_blank` links need `rel="noopener"` or `noopener noreferrer`; homepage nav list must be valid `<ul><li>…</li></ul>`.  
  **Rationale:** Valid HTML and accessibility.  
  **Implications:** Keep `p.Headings` inside list items because CSS depends on it.  
  **Affected:** homepage markup/CSS.  
  **Status:** Final.

- **Publications page should preserve grouped bibliography style.**  
  **Rationale:** Existing visual/citation style is part of the site’s scholarly identity.  
  **Implications:** Keep year grouping, `BibEntry`, `counter_bib`, `test#writings`, note lines, and exact citation formatting where possible.  
  **Affected:** `src/pages/publications/index.astro`, future citation components.  
  **Status:** Final.

- **Use JavaScript/animation sparingly.**  
  **Rationale:** Core content should remain static, durable, and accessible.  
  **Implications:** Avoid SPA-style navigation, scrolljacking, and gimmicky interactions; respect `prefers-reduced-motion`.  
  **Affected:** future UI/interactivity work.  
  **Status:** Final.

## Tentative decisions

- **Keep old rich Cosmic Connections schema as a temporary benchmark.**  
  **Rationale:** It contains useful detailed modelling not yet fully reproduced by the factory.  
  **Implications:** Do not delete until generated schema has matched/improved needed detail; do not use it as active production pattern.  
  **Affected:** old `src/data/schema/reviews/cosmic-connections/cosmic-connections.ts`, `src/lib/schema/review.ts`, `createReviewSchema.ts`.  
  **Status:** Tentative/temporary.

- **Publications page should use `CollectionPage` + `ItemList` JSON-LD.**  
  **Rationale:** Appropriate for a bibliography/index page.  
  **Implications:** Need a future `createPublicationsSchema.ts`; item IDs should prefer DOI, then local page, then stable fragment.  
  **Affected:** future `src/data/schema/publications/createPublicationsSchema.ts`, `/publications/`.  
  **Status:** Tentative but recommended.

- **Review local page schema may later split `WebPage` and `#review` article nodes.**  
  **Rationale:** Could provide more precise modeling between hosted page and scholarly article/manuscript.  
  **Implications:** Current `["ScholarlyArticle", "Review"]` local node is acceptable for now.  
  **Affected:** `createReviewSchema.ts`, review JSON-LD.  
  **Status:** Tentative refinement.

- **Use both `citation` and `isBasedOn` for local-to-published review relations where appropriate.**  
  **Rationale:** Captures relation between author manuscript/local version and published DOI version.  
  **Implications:** Needs consistent policy across reviews.  
  **Affected:** Review schema factory/frontmatter.  
  **Status:** Tentative.

- **Meaning of local review `datePublished` is not finalized.**  
  **Options:** Local site publication date, journal online publication date, or issue publication date.  
  **Rationale:** Different interpretations affect schema accuracy.  
  **Implications:** Need a consistent convention in frontmatter and JSON-LD.  
  **Affected:** Review frontmatter, `createReviewSchema.ts`.  
  **Status:** Tentative/open.

- **Review frontmatter schema shape is not fully final.**  
  **Open points:** Author/editor object vs array, `sameAs` string vs array, journal/volume/issue shape, `version`, reuse-policy fields, DOI normalization.  
  **Rationale:** Needs to support multiple publication types without overcomplication.  
  **Implications:** Check `src/content.config.ts` before adding many new files.  
  **Affected:** `src/content.config.ts`, all review Markdown, schema factory.  
  **Status:** Tentative.

- **Possible future extraction of publication/review components.**  
  **Decision under consideration:** Add components such as `ReviewListItem.astro`, `PublicationCitation.astro`, possibly `ReviewByline.astro`.  
  **Rationale:** Reduce duplication once content model stabilizes.  
  **Implications:** Check existing filename `ReviewByLine.astro` before imports; avoid premature refactor.  
  **Affected:** `src/components/`, `/publications/`, review pages.  
  **Status:** Tentative.

- **Thesis page implementation approach is undecided.**  
  **Decision known:** Slug is fixed.  
  **Open:** Static page vs content-driven thesis route; exact schema/page design.  
  **Rationale:** Thesis route may not yet exist but CV schema references it.  
  **Implications:** Need care before exposing schema links/sitemap entries.  
  **Affected:** future `src/pages/thesis/religious-atavism-climate-crisis/index.astro`, CV schema, sitemap.  
  **Status:** Tentative.

- **Seminars consolidation is undecided.**  
  **Options:** Keep separate `seminars` repo/subdomain or consolidate into main repo.  
  **Rationale:** Current GitHub Pages custom-domain behaviour is separate and nontrivial.  
  **Implications:** Consolidation could break relative links/assets or lose repo history if mishandled.  
  **Affected:** `seminars` repo, DNS/GitHub Pages settings, main site nav.  
  **Status:** Tentative/open.

- **Deployment workflow is not finalized/recorded.**  
  **Options:** GitHub Actions build, committed `dist/`, or other Pages workflow.  
  **Rationale:** Current production deployment method is unclear.  
  **Implications:** Must verify CNAME/custom-domain handling and sitemap/favicons/PDF output before deployment.  
  **Affected:** GitHub Pages settings, Actions, `dist/`, `public/CNAME` if used.  
  **Status:** Tentative/open.

- **Homepage/site title needs deliberate final choice.**  
  **Known variants:** `Re: Stevan Veljkovic`, `Dr Stevan Veljkovic – theorist and editor`, `S. Veljkovic – theorist and editor`, etc.  
  **Rationale:** Title affects browser display, SEO, and site identity.  
  **Implications:** Standardize in `site.ts` and page metadata.  
  **Affected:** `src/data/site.ts`, `SeoHead.astro`, homepage.  
  **Status:** Tentative/open.

- **Safari homepage name visibility fix is probable but unconfirmed.**  
  **Decision candidate:** Set `h1.Name { color: #fff; }`.  
  **Rationale:** Safari appeared to render black text on dark background.  
  **Implications:** Needs inspection in Safari before declaring fixed.  
  **Affected:** `src/styles/global.css`, homepage.  
  **Status:** Tentative/unresolved.
